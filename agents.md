# AI Agent Instructions

These instructions define how the AI should generally work in this project.

## Core Guidelines

- Prefer TypeScript when it is sensible for the task and ecosystem.
- Be thorough, but prioritize fast, incremental progress.
- Update documentation when behavior, APIs, setup steps, or refactors change project usage.
- Run the most relevant checks first (build/tests/lint), then broader checks as needed, and clearly report failures.
- Write code that is simple, readable, and easy for another AI to understand.
- Follow existing project patterns, naming, folder structure, and component conventions.
- Avoid assumptions from prior chats unless explicitly confirmed in the current context.
- Ask the human questions if something is not clear.
- Never edit generated or vendor files (for example dependency folders) unless explicitly asked.
- Keep a performance and accessibility baseline: responsive UI, semantic structure, and keyboard-friendly interactions.
